# Token Usage & Cost Tracking Guide

## Overview

The system now tracks token usage and calculates costs for all LLM calls across all agents. This provides visibility into:
- Total number of LLM calls per run
- Token usage (prompt, completion, total)
- Cost per agent and total cost
- Detailed breakdown by agent and call type

## How It Works

### 1. Token Tracker Module (`src/agents/token_tracker.py`)

A thread-safe tracking system that:
- Records every LLM call with token counts and cost
- Uses LiteLLM's built-in `completion_cost()` function for accurate cost calculation
- Aggregates usage by agent
- Provides summary statistics

### 2. Integration Points

Token tracking is automatically added to all LLM calls in:
- **IngestionAgent**: Schema discovery calls (`schema_discovery`)
- **LinkedInAnalyticsAgent**: Insight generation (`insight_generation`)
- **InstagramAnalyticsAgent**: Insight generation (`insight_generation`)
- **WebsiteAnalyticsAgent**: Insight generation (`insight_generation`)
- **StrategyAgent**: Strategy generation (`strategy_generation`)

### 3. Cost Calculation

The system uses LiteLLM's built-in cost calculation:
```python
cost = litellm.completion_cost(completion_response=response, model=model)
```

If LiteLLM's cost calculation is unavailable, it falls back to manual calculation using default pricing estimates.

## Where to View Token Usage

### In Streamlit Dashboard

After data ingestion completes, you'll see a **"ğŸ’° Token Usage & Cost Summary"** expander that shows:

1. **Summary Metrics** (4 columns):
   - Total LLM Calls
   - Total Tokens
   - Prompt / Completion tokens breakdown
   - Total Cost

2. **Breakdown by Agent** (table):
   - Agent name
   - Number of calls
   - Total tokens used
   - Cost per agent

### In Execution Summary

Token usage is also included in the execution summary displayed after ingestion:
- Total calls, tokens, and cost
- Breakdown by agent

## Programmatic Access

### Get Token Summary

```python
from src.agents.token_tracker import get_tracker

tracker = get_tracker()
summary = tracker.get_summary()

print(f"Total calls: {summary.total_calls}")
print(f"Total tokens: {summary.total_tokens:,}")
print(f"Total cost: ${summary.total_cost:.4f}")
```

### Get Detailed Breakdown

```python
breakdown = tracker.get_detailed_breakdown()
# Returns list of dicts with details for each call
```

### Export to Dictionary

```python
export_data = tracker.export_to_dict()
# Returns complete tracking data as dictionary
```

## Token Tracking Per Agent

Each agent's calls are tracked with:
- **Agent Name**: e.g., "LinkedInAnalyticsAgent"
- **Call Type**: e.g., "insight_generation", "schema_discovery"
- **Token Counts**: prompt_tokens, completion_tokens, total_tokens
- **Cost**: Calculated cost in USD
- **Timestamp**: When the call was made
- **Model**: Model used (e.g., "hackathon-gemini-2.5-pro")

## Reset Tracking

The token tracker is automatically reset at the start of each ingestion run. You can also manually reset:

```python
from src.agents.token_tracker import get_tracker
get_tracker().reset()
```

## Thread Safety

The tracker is thread-safe and works correctly with parallel agent execution (ThreadPoolExecutor). All tracking operations use locks to prevent race conditions.

## Notes

- **Cached Calls**: Schema discovery calls that use cached results don't make LLM calls, so they won't be tracked (no cost)
- **Thread Safety**: The tracker works correctly with parallel agent execution
- **Automatic Reset**: Tracking is reset at the start of each ingestion run
- **Cost Accuracy**: Uses LiteLLM's built-in cost calculation for the most accurate pricing

## Customization

To adjust pricing for your model, you can modify the fallback pricing in `token_tracker.py` if LiteLLM's cost calculation doesn't work for your setup.
